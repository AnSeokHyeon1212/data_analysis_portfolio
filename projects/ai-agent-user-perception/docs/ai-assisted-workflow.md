# AI-Assisted Workflow

이 프로젝트는 AI를 코딩과 문서화의 보조 도구로 활용했습니다. 분석을 전부 수작업으로 작성했다고 표현하지 않으며, 도구가 맡은 부분과 본인이 판단한 부분을 구분합니다.

## Researcher Decisions

- 연구 질문을 Google Play 리뷰 데이터로 구체화
- 데이터 수집 범위와 비교 기준일 설정
- 전처리·제외 기준과 분석 변수 선택
- VADER를 감성 집단 구분에 사용하는 이유 결정
- 감성별 통합 LDA와 하위 LDA 설계
- 대표 키워드와 문서를 검토해 토픽명 확정
- 결과의 서비스 운영 의미와 한계 검토
- 발표 자료 제작과 학회 발표

## AI-Assisted Tasks

- 반복 코드 초안 작성과 함수화
- 오류 원인 탐색과 수정안 제안
- 실행 시간·메모리 최적화 아이디어
- 시각화 코드와 레이아웃 초안
- 대표 키워드와 문장을 바탕으로 한 토픽명 후보
- 결과 해석 문장의 초안과 표현 교정

## Verification Process

1. AI가 제안한 코드를 데이터 구조와 분석 목적에 맞게 수정했습니다.
2. 중간 데이터의 행 수, 결측, 기간 구분과 감성 분포를 확인했습니다.
3. 토픽 키워드와 대표 문서를 함께 읽고 이름을 검토했습니다.
4. 코드에서 산출한 CSV와 발표 자료의 수치를 대조했습니다.
5. 인과관계로 오해할 수 있는 표현을 전후 연관성 수준으로 수정했습니다.

AI 활용 여부보다 중요한 것은 **분석 선택을 설명하고 결과를 검증할 수 있는가**라고 생각합니다. 이 문서는 그 책임 범위를 명확히 하기 위해 공개합니다.
