# Methodology

## 1. Research Design

Agent 출시일인 2025-07-17을 기준으로 리뷰를 `before`와 `after`로 구분했습니다. 기준일 당일은 출시 전후 노출 여부를 판별하기 어려워 비교에서 제외했습니다.

이 분석의 목표는 평균 평점 차이가 아니라 **사용자가 언급하는 주제의 구성 변화**를 확인하는 것입니다. 따라서 변화량은 각 토픽 확률의 기간별 평균 차이 `after - before`로 계산했습니다.

## 2. Preprocessing

1. 최신 누적 원시 CSV 선택
2. 날짜·평점·리뷰 기준 중복 제거
3. 작성자 이름 제거와 내부용 `review_id` 생성
4. 언어 탐지 확률, 라틴 문자 비율, 영어 어휘 비율을 이용한 영어 리뷰 필터링
5. URL·이메일·불필요 문자를 정규화하고 저정보 리뷰 제거
6. spaCy로 명사 중심 토큰을 추출하고 표제어 처리

전처리 기준과 제거 건수는 실행 시 audit table로 저장합니다. 제거된 리뷰 원문 로그는 공개하지 않습니다.

## 3. Sentiment Segmentation

NLTK의 VADER compound score를 사용했습니다.

- Positive: `compound >= 0.20`
- Neutral: `-0.20 < compound < 0.20`
- Negative: `compound <= -0.20`

VADER를 선택한 이유는 감성분석 정확도 자체를 연구 대상으로 삼은 것이 아니라, 토픽 분석 전 리뷰 집단을 일관된 규칙으로 나누기 위해서입니다. 리뷰 특화 Transformer와의 성능 비교는 수행하지 않았으므로 감성 라벨을 정답으로 해석하지 않습니다.

## 4. Integrated LDA

출시 전·후에 별도 LDA를 학습하면 두 모델의 토픽 번호가 일치한다는 보장이 없습니다. 이를 피하기 위해 감성 집단별로 전후 문서를 합친 하나의 LDA를 학습했습니다.

1. 긍정과 부정 리뷰를 각각 분리
2. 전후에서 균형 표본을 추출해 `k` 후보의 coherence 비교
3. 전체 감성 집단에서 최종 LDA 학습
4. 각 문서의 토픽 확률 계산
5. 기간별 평균 토픽 확률과 변화량 계산

`k=5~14` 후보를 검토한 기존 학회 분석에서 긍정과 부정 모두 `k=7`의 coherence가 가장 높았습니다.

| 감성 | 선택 k | coherence |
|---|---:|---:|
| Positive | 7 | 0.3656 |
| Negative | 7 | 0.3986 |

## 5. Topic Naming

토픽 이름은 다음 근거를 함께 확인해 결정했습니다.

- 토픽별 상위 가중치 키워드
- 토픽 확률이 높은 출시 전·후 대표 문서
- 비슷한 의미의 키워드가 한 토픽에서 반복되는지
- 서비스 운영 이슈로 설명할 수 있는 일관성이 있는지

Claude는 토픽명 후보와 해석 초안을 제안하는 데 사용했고, 최종 이름은 대표 키워드와 문서를 다시 확인한 뒤 연구자가 결정했습니다.

## 6. Sub-LDA

변화량이 큰 부정 토픽 안에서 세부 이슈를 확인하기 위해 상위 토픽 확률이 기준값 이상인 문서를 선별하고 하위 LDA를 학습했습니다. 하위 토픽도 전후 문서를 같은 토픽 공간에서 비교했습니다.

## 7. Reproducibility Notes

- 난수 시드를 고정합니다.
- 토픽 수 탐색 결과, 토픽 키워드, 기간별 비중과 변화량을 CSV로 저장합니다.
- 원본 데이터와 원문 포함 산출물은 Git에서 제외합니다.
- 공개 노트북은 기존 결과 CSV나 모델을 입력으로 사용하지 않고 원시 리뷰에서 시작하도록 설계했습니다.
